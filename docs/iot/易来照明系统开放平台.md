# 易来照明系统开放平台

# 1. 简介

本平台专为三方厂商打造，旨在提供一个高效、智能的照明解决方案。平台集成了多种功能，允许厂商轻松拉取家庭/项目中的各类数据，并实现对设备的远程控制和实时监控。通过开放的平台架构，厂商可以灵活对接不同品牌和类型的照明设备，满足不同客户的个性化需求。

平台的核心优势包括：

1. **数据集成与分析**：从家庭/项目中获取照明相关的数据，帮助厂商分析用户的使用习惯，并据此优化产品设计和功能。
2. **设备控制与管理**：支持对接不同的智能照明设备，提供统一的控制接口，让厂商能够方便地管理多个设备，提升用户体验。
3. **实时监控与反馈**：通过实时监控设备的运行状态，及时反馈故障信息，减少运维成本，提升设备的稳定性。
4. **开放 API 接口**：提供丰富的 API 接口，便于第三方应用和服务的无缝集成，增强平台的兼容性和扩展性。

无论是针对家庭/项目、办公场所，还是大型商业空间，平台都能为厂商提供全方位的照明解决方案，帮助他们快速响应市场需求，提升产品的市场竞争力。

# 2. 快速接入

## 2.1 OAuth 对接

Yeelight IoT 开放平台采用 [OAuth2.0](http://www.ruanyifeng.com/blog/2019/04/oauth_design.html) 进行授权，OAuth 2.0 是目前最流行的授权机制，用来授权第三方应用，获取用户数据。

在回复邮件中我们会附带 OAuth 授权所需的 client_id 以及 client_secret 信息，如果你已经获得了这些信息那么我们开始吧。

### 2.1.1 获取授权码

请求地址: [https://api.yeelight.com/apis/account/oauth/authorize](https://api.yeelight.com/apis/account/oauth/authorize)

请求方法: GET

请求参数:

| 参数名称<br/>      | 必须<br/> | 类型<br/>    | 备注<br/>                                                                                                               |
| ------------------ | --------- | ------------ | ----------------------------------------------------------------------------------------------------------------------- |
| client_id<br/>     | 是<br/>   | string <br/> | Yeelight 分配的应用 ID，可以联系管理员申请<br/>                                                                         |
| redirect_uri<br/>  | 是<br/>   | string<br/>  | 回调地址, 必须和管理员给你添加应用时设置的一致(参数部分可不一致)<br/>                                                   |
| response_type<br/> | 是<br/>   | string<br/>  | 描述获取授权的方式， 这里 response_type=code<br/>                                                                       |
| scope<br/>         | 否<br/>   | string<br/>  | 申请的接口权限 ，可以传递多个，用空格分隔，目前未启用，可以填 read write<br/>                                           |
| state<br/>         | 否<br/>   | string<br/>  | 随机字符串，授权请求成功后原样返回，该参数用于防止 CSRF 攻击，强烈建议传递该参数<br/>                                   |
| skip_confirm<br/>  | 否<br/>   | string<br/>  | 默认值为 true，授权有效期内的用户在已登录情况下，不显示授权页面，直接通过。如果需要用户每次手动授权，设置为 false <br/> |

返回值:

- 成功响应

如果授权成功，授权服务器会将用户的浏览器重定向到 `redirect_uri`，并带上相关参数：

```go
http://example.com/example?code=CODE&state=STATE
```

返回值说明：

| 参数名称<br/> | 必须<br/> | 类型<br/>   | 备注<br/>                                         |
| ------------- | --------- | ----------- | ------------------------------------------------- |
| code<br/>     | 是<br/>   | string<br/> | 用来换取 access_token 的授权码，只能使用一次<br/> |
| state<br/>    | 否<br/>   | string<br/> | 如果请求时传递参数，会回传该参数<br/>             |

- 失败响应

如果授权失败，授权服务器会将用户的浏览器重定向到 `redirect_uri`，并带上相关参数：

```go
http://example.com/example?error=ERROR&error_description=ERROR_DESCRIPTION&state=STATE
```

返回值说明：

| 参数名称<br/>          | 必须<br/> | 类型<br/>   | 备注<br/>                             |
| ---------------------- | --------- | ----------- | ------------------------------------- |
| error<br/>             | 是<br/>   | int<br/>    | 错误码见下方定义<br/>                 |
| error_description<br/> | 是<br/>   | string<br/> | 错误描述信息<br/>                     |
| state<br/>             | 否<br/>   | string<br/> | 如果请求时传递参数，会回传该参数<br/> |

#### 相关错误码定义

| **错误 CODE**<br/> | **错误信息**<br/>               | **详细描述**<br/>                                             |
| ------------------ | ------------------------------- | ------------------------------------------------------------- |
| 401<br/>           | invalid_client<br/>             | 应用不存在, 或者 client_id 和 client_secret 不匹配<br/>       |
| 400<br/>           | invalid_request<br/>            | 请求缺少必要参数，或者参数格式不正确<br/>                     |
| 403<br/>           | insufficient_scope<br/>         | 资源服务器处理请求时令牌中的作用域不足<br/>                   |
| 400<br/>           | invalid_grant<br/>              | grant_type 无效, grant type 支持 token 或者 code<br/>         |
| 401<br/>           | unauthorized_client<br/>        | 客户端没有权限使用该请求, 需要申请相应权限<br/>               |
| 400<br/>           | unsupported_grant_type<br/>     | grant_type 不被授权服务器所支持<br/>                          |
| 400<br/>           | invalid_scope<br/>              | scope 是无效的、未知的，或格式不正确的<br/>                   |
| 401<br/>           | invalid_token<br/>              | access_token 无效或已过期<br/>                                |
| 400<br/>           | Invalid refresh token<br/>      | refresh_token 无效或已经过期<br/>                             |
| 400<br/>           | redirect_uri_mismatch <br/>     | 回调地址与注册的值不匹配或者不是一个合法的 URI<br/>           |
| 400<br/>           | unsupported_response_type<br/>  | 响应类型不被授权服务器所支持<br/>                             |
| 400<br/>           | access_denied<br/>              | 用户或授权服务器拒绝了请求, 可能是用户取消了对应用的授权<br/> |
| 400<br/>           | Invalid authorization code<br/> | 授权码无效或已经过期, 授权码只能用一次<br/>                   |
| 400<br/>           | unauthorized_user<br/>          | 无效的用户<br/>                                               |

### 2.1.2 获取访问令牌

请求地址: [https://api.yeelight.com/apis/account/oauth/token](https://api.yeelight.com/apis/account/oauth/token)

请求方法: POST

请求参数:

| 参数名称<br/>      | 必须<br/> | 类型<br/>   | 备注<br/>                                                                                                                                             |
| ------------------ | --------- | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| client_id<br/>     | 是<br/>   | string<br/> | Yeelight 分配的应用 ID，可以联系管理员申请 <br/>                                                                                                      |
| redirect_uri<br/>  | 是<br/>   | string<br/> | 回调地址, 必须和管理员给你添加应用时设置的一致(参数部分可不一致) <br/>                                                                                |
| client_secret<br/> | 是<br/>   | string<br/> | Yeelight 分配的应用秘钥，可以联系管理员申请 <br/>                                                                                                     |
| grant_type<br/>    | 是<br/>   | string<br/> | 这里 grant_type=authorization_code<br/>                                                                                                               |
| code<br/>          | 是<br/>   | string<br/> | 授权码接口拿到的授权码，只能使用一次<br/>                                                                                                             |
| device<br/>        | 否<br/>   | string<br/> | 允许一个用户登录多个设备，且最多登录 10 个。改参数不传时，系统会随机生成一个。超出 10 个后，会按照授权时间，踢出最先授权的设备。强烈建议传该参数<br/> |

返回值:

- 成功响应

如果请求成功，授权服务器会返回 JSON 格式的字符串：

| 参数名称<br/>      | 必须<br/> | 类型<br/>    | 备注<br/>                                                                        |
| ------------------ | --------- | ------------ | -------------------------------------------------------------------------------- |
| access_token<br/>  | 是<br/>   | string<br/>  | 访问令牌<br/>                                                                    |
| token_type<br/>    | 是<br/>   | string <br/> | token 类型 <br/>                                                                 |
| refresh_token<br/> | 是<br/>   | string<br/>  | 刷新令牌<br/>                                                                    |
| expires_in<br/>    | 是<br/>   | long<br/>    | 访问令牌生命周期（单位：秒）<br/>                                                |
| scope<br/>         | 是<br/>   | string<br/>  | 访问令牌实际权限范围<br/>                                                        |
| id<br/>            | 是<br/>   | long<br/>    | 当前授权用户的 ID<br/>                                                           |
| region<br/>        | 是<br/>   | string<br/>  | 当前授权用户的地域<br/>                                                          |
| device<br/>        | 是<br/>   | string<br/>  | 当前授权的用户设备，请求参数中传了的话，会原样返回，否则返回 统会随机生成的<br/> |
| client_id<br/>     | 是<br/>   | string<br/>  | 当前授权应用的 ID<br/>                                                           |
| username<br/>      | 是<br/>   | string <br/> | 当前授权用户的用户名<br/>                                                        |
| jti<br/>           | 是<br/>   | string<br/>  | jwt-token-id，token 的唯一属性<br/>                                              |

## 2.2 开始调用

### 2.2.1 协议简介

Yeelight 开放平台 api 统一使用 Post 请求，请求信息包括 Http Header、Http body 两部分。

1. Http Header：包含接入方标识、Oauth token

| 参数名<br/>        | 说明<br/>                                                            |
| ------------------ | -------------------------------------------------------------------- |
| authorization<br/> | 通过 Yeelight OAuth 获取的 token 值 eg: Bearer V3_3hF4q**\*\***<br/> |
| clientId<br/>      | 接入开放平台时分配的接入方 key<br/>                                  |

1. 返回值：包括了状态(code)、状态描述(msg)以及具体数据(data)

```go
{
    "code": 200,
    "msg": "success",
    "data": {
        /** 已省略 **/
    }
}
```

如果是列表数据，data 中为 page 结构，result 中为 row 数组

```json
{
  "total": 20,
  "pageNum": 2,
  "pageSize": 10,
  "result": []
}
```

### 2.2.2 概念

#### 空间

家庭/项目：家庭/项目是最大的空间单元，不支持跨家庭/项目的控制操作。
房间：房间是最小的空间单元，类似于现实生活中的房间概念。
区域：区域是对家庭/项目内部空间的进一步划分，通常由一个或多个房间组成。

#### 节点

空间节点：包括家庭/项目、房间和区域三类节点，代表不同层级的空间结构。
设备节点：指位于房间中的每个物理设备，作为控制的基本单元。
设备组节点：设备可以进行编组，通过组控功能实现统一控制。

#### 情景

- 情景是多个行为的集合，包含了对一个或多个节点的不同指令操作。

#### 行为

- 每个节点都有所支持的行为，我们可以对节点下发行为指令来进行对应的控制。
- 指令包含：开关、设置亮度绝对值、增加亮度、降低亮度、设置色彩/色温绝对值、开合、应用情景。

## 3.3 事件通知

为了实现高效的实时通信，我们提供了稳定的 WebSocket 服务。通过该服务，客户端可以与服务器建立长连接，实现事件信息的实时同步。以下是该服务的关键特点和使用方式：

1. **实时通信**：通过 WebSocket 协议，客户端与服务器之间能够保持持续的长连接，确保事件信息能够第一时间从服务器同步到客户端。
2. **高效连接**：服务支持轻量级、低延迟的通信方式，极大地提高了数据传输的速度和效率，适合需要即时响应的应用场景。
3. **统一地址接入**：客户端只需使用以下格式的 URL 即可连接至服务器：

```bash
wss://push.yeelight.com/ws/{token}
```

1. 其中，`{token}` 是客户端的 token 信息（不含"Bearer "前缀)，确保每个客户端连接的安全性和专属性。
2. **灵活适用**：该服务适用于各种需要实时数据推送和即时响应的应用场景，提供稳定的消息传输保障。

无论是消息通知、状态更新，还是设备控制，借助我们的 WebSocket 服务，您的应用可以更加流畅、可靠地与服务器保持互动，为终端用户提供卓越的使用体验。

### 3.3.1 消息订阅

在 WebSocket 连接建立后，客户端需要立即发送订阅请求，以便服务器能够推送相关的消息。在成功订阅之后，客户端将开始接收实时消息推送。这一过程通常在连接建立后的第一时间完成，以确保消息能够及时到达客户端。

具体消息格式如下：

```json
{
  "id": 1,
  "method": "subscribe",
  "params": {
    "type": 2
  },
  "timestamp": 1722133937,
  "version": "1.0"
}
```

其中，`id` 字段为自增 id，`method` 字段标识消息类型为订阅，`params.type` 字段固定为 2，`timestamp` 字段表示发送心跳的时间戳，`version` 字段为协议版本号，目前为 1.0。

### 3.3.2 心跳保持

为了保持 WebSocket 连接的稳定性，客户端需要定时发送心跳消息。心跳的格式如下：

- **心跳间隔**：每隔固定时间发送心跳（通常建议每隔 20 秒发送一次）。
- **心跳格式**：心跳消息应采用以下格式：

```json
{
  "id": 2,
  "method": "heartbeat",
  "timestamp": 1722133937,
  "version": "1.0"
}
```

其中，`id` 字段为自增 id，`method` 字段标识消息类型为心跳，`timestamp` 字段表示发送心跳的时间戳，`version` 字段为协议版本号，目前为 1.0。

如果服务器在 1 分钟内未收到心跳消息，将自动断开连接，以确保连接的可靠性。

### 3.3.3 属性变更推送

```json
{
  "type": "prop",
  "msgId": "d29fa2375b374d7ba941bb859c9ca0d2",
  "nodes": [
    {
      "id": 228215,
      "nt": 2,
      "params": {
        "p": true,
        "ct": 4500,
        "l": 100,
        "o": true
      }
    }
  ],
  "timestamp": 1724658984,
  "version": "1.0"
}
```

其中包含了发生属性变更的节点信息。消息的关键字段如下：

- **nodes**: 这是一个包含变更节点的列表。每个节点代表一个发生属性变更的具体对象。

  - **id**: 表示节点的唯一标识符（节点 ID）。
  - **nt**: 表示节点类型，用于区分不同类型的节点。
  - **params**: 一个属性映射（map），其中键为属性名称，值为相应的属性值，描述了该节点的具体变更内容。

- **type**: 描述执行的操作类型，这里为 `"prop"`，表示一个属性更新操作。
- **msgId**: 消息的唯一标识符，用于跟踪和识别特定的消息。
- **timestamp**: 消息生成的时间戳，用于记录变更的发生时间。
- **version**: 表示消息的版本号，确保兼容性和一致性。

其中包含了发生属性变更的节点信息。消息的关键字段如下：
